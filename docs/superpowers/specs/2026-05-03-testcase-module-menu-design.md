# 测试用例模块树与页面布局优化设计

## 1. 模块树结构优化

### 树节点结构
- 根节点「全部用例」— 固定，显示总用例数（含所有子模块），不可拖动/删除
- 首个固定子节点「未规划用例」— 不可删除/重命名/拖动，始终排在第一位，不可作为拖放目标
- 其他模块：用户手动创建，支持重命名、删除、拖动

### 展开/收起行为
- 「全部用例」右侧的展开收起按钮切换全局状态：
  - 收起：只显示「未规划用例」和第一级模块（根的直接子级）
  - 展开：递归展开所有层级的子模块
- 每个模块可以独立点击箭头展开/收起自身子级
- 点击模块名 → 选中并筛选该模块下的用例
- 展开收起操作与选中互不干扰

### 拖动排序
- 树节点启用 `draggable`，通过 `n-tree` 的拖拽事件处理
- 拖到另一模块上 → 成为其子模块（调用 `POST /module/update` 更新 parent_id）
- 拖到两模块之间 → 成为平级模块
- 后端的 `update_module_closure` 已有处理 parent_id 变更的逻辑

## 2. 用例数量显示（含子模块）

后端 `get_module_stats` 改为递归统计：从 `ModuleClosure` 表查询每个模块的所有后代，再汇总用例数。

每个模块显示其子树内的所有用例总数。

## 3. 批量操作更新模块

后端新增 `POST /testcase/batch_update_module`，接收 `{ ids: [...], module_id: int }`。
前端新增「批量修改模块」选项，弹出树选择器选目标模块。

## 4. 页面布局调整

所有测试管理页面统一调整：项目选择器移入筛选栏与其余筛选条件同行，操作按钮（新建、导入、批量操作）置于筛选栏下方、表格上方。

涉及页面：testcase, testplan, defect, file, msgconfig
